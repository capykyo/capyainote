# Directus 概览

> 文档来源：https://directus.io/docs/getting-started/overview

## 是什么

Directus 是一个**后端平台**，连接到你的数据库和外部服务，同时提供：

- **Data Engine**：为开发者提供 API 和工具，用于程序化数据访问
- **Data Studio**：为团队提供可视化数据管理 Web 应用

核心理念：Directus 不替换你的数据库，而是**包裹**在现有数据库之上，让数据库保持原生结构，随时可以脱离 Directus 直接访问。

---

## 核心架构

```
┌─────────────────────────────────────────┐
│              你的应用 / 前端              │
└────────────────┬────────────────────────┘
                 │ REST / GraphQL / SDK
┌────────────────▼────────────────────────┐
│           Data Engine（后端）            │
│  • 自动生成 API                          │
│  • 用户认证与权限管理                    │
│  • 实时订阅（WebSocket）                 │
│  • 自动化流程（Flows）                   │
│  • Webhooks                             │
└────────────────┬────────────────────────┘
                 │
┌────────────────▼────────────────────────┐
│              你的数据库                  │
│  PostgreSQL / MySQL / SQLite / 等        │
└─────────────────────────────────────────┘
                 ▲
┌────────────────┴────────────────────────┐
│           Data Studio（前端）            │
│  • 内容管理界面                          │
│  • 文件管理                             │
│  • Insights 数据看板                    │
│  • 角色与权限配置                        │
└─────────────────────────────────────────┘
```

## 核心概念

| 概念 | 说明 |
|------|------|
| **Collection** | 对应数据库中的一张表 |
| **Field** | 对应表中的一列（字段） |
| **Item** | 对应表中的一行（记录） |
| **Role** | 用户角色，控制权限范围 |
| **Permission** | 基于角色的字段级读写权限 |
| **Flow** | 事件/定时触发的自动化工作流 |
| **Extension** | 自定义插件，扩展功能 |

### 关系类型

| 类型 | 说明 | 示例 |
|------|------|------|
| M2O（多对一） | 多条记录指向一条 | 文章 → 作者 |
| O2M（一对多） | 一条记录关联多条 | 作者 → 文章列表 |
| M2M（多对多） | 双向多条关联 | 文章 ↔ 标签 |
| M2A（多对任意） | 关联不同 Collection 的记录 | 页面块（灵活内容） |

## 主要使用场景

- **Headless CMS**：通过 API 向任意前端（Web/App）分发内容
- **Backend as a Service**：快速获得 CRUD API、认证、权限，无需手写后端
- **内部工具**：搭建管理后台、数据看板，无需编码
- **数据管理平台**：为团队提供统一的数据操作界面

## 部署方式

| 方式 | 说明 |
|------|------|
| **Directus Cloud** | 官方托管，90 秒启动，免运维 |
| **自托管（Self-Hosted）** | 部署在自己的服务器，完全掌控数据和配置 |

### 商用授权（重要）

Directus 使用 **BSL 1.1** 许可证：

- 总融资/营收 **低于 500 万美元**：可免费商用，无需额外授权
- 总融资/营收 **超过 500 万美元**：需联系官方申请商业授权

---

## 自托管技术要求

运行生产级 Directus 实例，需要具备：

- Docker / Docker Compose
- 数据库：**PostgreSQL**（推荐）、MySQL、SQLite 等
- Redis（缓存与 Session）
- 文件存储：本地磁盘 / S3 / 云存储
- 反向代理：Nginx / Caddy（HTTPS）
- 监控、备份方案

---

## 技术栈

- 后端：**Node.js**
- 数据库 ORM：**Knex.js**
- 前端（Data Studio）：**Vue 3**
- API：REST + GraphQL 双支持
- 实时：WebSocket
